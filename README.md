# WebFactory

> **WebFactory: Automated Compression of Foundational Language Intelligence into Grounded Web Agents**
> Sicheng Fan, Qingyun Shi, Shengze Xu, Shengbo Cai, Tieyong Zeng, Li Ling, Yanyi Shang, Dehan Kong
> ICLR 2026 · [arXiv:2603.05044](https://arxiv.org/abs/2603.05044)

WebFactory is an automated reinforcement-learning framework that **compresses a large language model's latent web knowledge into a lightweight, deployable web agent** — without live web access and without human-annotated trajectories.

The repo contains two complementary components:

| Component | Location | Purpose |
|---|---|---|
| **WebFactory Website Suite** | `src/` | 10 realistic offline websites used as training & evaluation environments |
| **open operator** | `open operator/` | LLM browser-agent engine: DOM + visual operator modes, RAG memory, RL reward, evaluation pipeline |

---

## Why WebFactory?

Training GUI agents on live websites is slow, expensive, and hard to reproduce. Human trajectory annotation is even costlier. WebFactory solves both problems with a closed-loop pipeline:

1. **Automated environment creation** — 10 fully functional offline websites covering e-commerce, travel, job search, events, food delivery, and more.
2. **Knowledge-informed task generation** — tasks are generated by querying the LLM's own world knowledge about each website.
3. **LLM-driven trajectory collection** — a frontier LLM (GPT-4o / Claude) operates the browser and records full interaction traces.
4. **RL training** — the collected trajectories train a smaller model (e.g., 3B parameters) with a decomposed reward `Rt = α·Rf + β·Raccuracy`.
5. **Evaluation** — the trained agent is evaluated on Online-Mind2Web and the same 10 websites using Task Completion Rate (TCR).

> Key result: **WebFactory-3B achieves 71.8% Operational TCR**, matching agents trained on human-annotated data from much larger datasets, and demonstrates **53.4% offline-to-online transfer**.

## open operator — LLM Browser Agent

The `open operator` folder is the agent engine used to collect training trajectories and run evaluations. It supports two observation modes:

- **DOM mode**: parses the accessibility tree into structured text for LLM planning.
- **Operator mode**: passes live browser screenshots directly to OpenAI's `computer-use-preview` model, which returns pixel-coordinate actions — no DOM parsing required.

### Setup

```bash
cd "open operator"

# Create Python environment
conda create -n webfactory python=3.11
conda activate webfactory
pip install -r requirements.txt

# Install Playwright browser
python -m playwright install chromium

# Node.js dependencies (for Google Search action)
npm init -y && npm install axios
```

### API Keys

```bash
# Required for Operator mode and reward models
export OPENAI_API_KEY=your_openai_key

# Optional: other LLM backends
export ANTHROPIC_API_KEY=your_anthropic_key
export GOOGLE_API_KEY=your_google_key
export TOGETHER_API_KEY=your_together_key

# Optional: Google Custom Search (for GOOGLE_SEARCH action)
export GOOGLE_CX=your_custom_search_engine_id

# Optional: custom log directory (default: test/exp/logs)
export LOG_DIR=my_logs/
```

See [`agent/LLM/README.md`](open%20operator/agent/LLM/README.md) for per-provider setup details.

### Running Evaluations

**DOM mode — single task:**
```bash
xvfb-run -a python eval.py \
  --observation_mode dom \
  --global_reward_mode dom_reward \
  --global_reward_text_model gpt-4o-mini \
  --planning_text_model gpt-4.1 \
  --single_task_name "Find the cheapest economy flight from NYC to LA" \
  --single_task_website "http://localhost:3000/flights"
```

**Operator mode — single task:**
```bash
xvfb-run -a python eval_op.py \
  --observation_mode operator \
  --global_reward_mode dom_reward \
  --global_reward_text_model gpt-4o-mini \
  --planning_text_model computer-use-preview-2025-03-11 \
  --single_task_name "Search for hotels in Paris for 2 adults, check-in July 1" \
  --single_task_website "http://localhost:3000/hotels" \
  --rag_logging_enabled \
  --rag_log_dir test/exp/rag_logs
```

**Batch evaluation (Operator mode with RAG + End Judge):**
```bash
xvfb-run -a python batch_eval_op.py \
  --global_reward_mode dom_reward \
  --global_reward_text_model gpt-4.1 \
  --snapshot test/exp \
  --output_log test/exp/batch_operator_log.txt \
  --rag_mode vision_rag \
  --rag_logging_enabled \
  --rag_log_dir test/exp/rag_logs \
  --prompt_logging_enabled \
  --prompt_log_dir test/exp/prompt_logs \
  --end_judge enabled \
  --end_judge_confidence_threshold 0.8 \
  --end_judge_min_steps 2 \
  --consecutive_error_threshold 2
```

### Standalone Computer Use Demo

`operator_demo.py` is a self-contained script demonstrating the OpenAI Computer Use Responses API. It launches a headless Chromium browser, sends the task to the model, and saves screenshots + a full JSON trace.

```bash
python operator_demo.py \
  --task "Search for 2025 AI tools news and click on the first OpenAI result" \
  --out-dir ./demo_screenshots \
  --json-out ./demo_trace.json
```

Output:
- `demo_screenshots/step_001.png`, `step_002.png`, … — screenshot at each action step
- `demo_trace.json` — full response trace with actions and the final answer

### Processing Evaluation Results

After a batch run, parse the logs into structured JSON and build a trajectory dataset:

```bash
# Operator mode
python utils/operator_parser.py
python utils/operator_dataset_process.py \
  --results_dir results/ \
  --output_dir dataset/

# DOM mode
python utils/parser.py
python utils/dataset_process.py
```

Processed dataset layout:
```
results/
└── <task_id>/
    ├── trajectory/
    │   ├── step_0_<timestamp>.png
    │   └── step_N_<timestamp>.png
    └── result.json
```

## Paper Summary

**Core idea — "Intelligence Compression Factory":** frontier LLMs act as *architects of their own embodiment*. They generate tasks, collect trajectories, and teach a smaller student model through RL — transferring web knowledge without any human labels.

**System components:**
- **Automated Task Generator**: queries the LLM's knowledge of each website to produce realistic tasks.
- **Trajectory Collector**: a GPT-4o / Claude agent operates the offline websites and records full interaction traces (screenshots + actions).
- **RL Trainer**: trains a 3B student with decomposed reward `Rt = α·Rf + β·Raccuracy`.
- **RAG Memory**: VLM2Vec-based retrieval augments the agent with similar historical trajectories at inference time.
- **Embodiment Potential**: a new metric measuring how well a base LLM's latent knowledge transfers to grounded agent behavior.

**Results:**
- WebFactory-3B: **71.8% Operational TCR** on the WebFactory website suite.
- **53.4% offline-to-online transfer** — knowledge learned on synthetic offline data generalises to live websites.
- Matches human-annotated baselines with zero human trajectory annotation.
